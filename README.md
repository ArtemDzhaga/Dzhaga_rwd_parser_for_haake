# Локальный CLI для HAAKE RheoWin `.rwd`

Проект делает одну главную вещь:

на вход получает один `.rwd` файл, несколько `.rwd` файлов или папку с `.rwd`,
а на выходе создает единую Excel-книгу `.xlsx` с табличными данными измерений и
метаданными прибора.

Основной файл проекта:

```bash
haake_cli.py
```

## Быстрый запуск

Перейти в папку проекта:

```bash
cd /Users/artem/Desktop/Output/iam_ras/haake_parser
```

Обработать один файл:

```bash
python3 haake_cli.py "32th_pein_15_steps_60C_1hz_max_ v2.rwd" -o result.xlsx
```

Обработать несколько файлов:

```bash
python3 haake_cli.py "file_1.rwd" "file_2.rwd" "file_3.rwd" -o result.xlsx
```

Обработать все `.rwd` в папке:

```bash
python3 haake_cli.py "/путь/к/папке/с/rwd" -o result.xlsx
```

Обработать все `.rwd` в папке и вложенных папках:

```bash
python3 haake_cli.py "/путь/к/папке/с/rwd" --recursive -o result.xlsx
```

Открыть окно выбора папки:

```bash
python3 haake_cli.py --gui -o result.xlsx
```

Добавить диагностический лист с исходными числовыми блоками:

```bash
python3 haake_cli.py "/путь/к/папке/с/rwd" --recursive --include-diagnostics -o result.xlsx
```

## Drag and drop

Можно использовать drag and drop прямо в терминале.

Сначала набрать:

```bash
python3 haake_cli.py 
```

Потом перетащить один `.rwd`, несколько `.rwd` или папку в окно терминала и
нажать Enter.

Пример после перетаскивания:

```bash
python3 haake_cli.py /Users/artem/Desktop/experiments/sample_1.rwd /Users/artem/Desktop/experiments/sample_2.rwd
```

## Что будет в Excel

Скрипт создает одну книгу `.xlsx`.

### `data`

Главный лист с табличными данными измерений.

Одна строка - одна точка измерения.

Основные колонки:

- `file_name` - имя исходного `.rwd`;
- `point` - номер точки;
- `time_s` - время точки, с;
- `time_start_s` - вспомогательное время начала/сегмента, с;
- `tau_pa` - напряжение `Tau`, Па;
- `gamma` - деформация;
- `complex_modulus_pa_est` - расчетная оценка `Tau/gamma`, Па;
- `frequency_hz` - частота, Гц;
- `temperature_c` - температура, градусы Цельсия;
- `temperature_k` - температура, Кельвины;
- `gamma_aux` - дополнительный канал деформации/амплитуды;
- `signal_aux` - дополнительный числовой канал RheoWin.

Для нескольких файлов все строки лежат в одном листе `data`, отсортированы по
имени файла. В Excel включен автофильтр.

### `instrument_metadata`

Метаданные прибора в столбчатом представлении.

Первая колонка - название поля. Каждая следующая колонка - отдельный `.rwd`
файл. Если файлов несколько, они расположены в алфавитном порядке.

Поля:

- `file_path`;
- `size_bytes`;
- `format`;
- `rheowin_version`;
- `date`;
- `time`;
- `instrument_model`;
- `device`;
- `geometry`;
- `temperature_controller`;
- `gap`;
- `serial_numbers`;
- `driver_versions`;
- `firmware_versions`;
- `job_file`;
- `mode`;
- `measurement_points_extracted`.

### `logs`

Служебные строки RheoWin:

- старт и завершение элементов методики;
- autosave;
- сообщения выполнения;
- остановка прибора;
- сохранение сегментов.

### `numeric_blocks`

Появляется только с флагом `--include-diagnostics`.

Это технический лист: он показывает, какие бинарные числовые блоки были найдены
внутри `.rwd`, их коды и первые значения. Он нужен для проверки и дальнейшего
уточнения соответствия каналов RheoWin.

## Как работает процесс внутри проекта

1. Пользователь запускает `haake_cli.py` и передает путь к файлу, нескольким
   файлам или папке.

2. Скрипт собирает список `.rwd` файлов:
   - если передан файл, берет его;
   - если передана папка, ищет внутри нее `.rwd`;
   - если указан `--recursive`, ищет также во вложенных папках.

3. Список файлов сортируется по алфавиту.

4. Каждый `.rwd` читается как бинарный файл.

5. Из файла извлекаются читаемые текстовые блоки RheoWin. Из них берутся:
   - дата;
   - время;
   - версия RheoWin;
   - модель прибора;
   - геометрия;
   - gap;
   - серийные номера;
   - firmware/driver;
   - путь к `.rwj` методике;
   - режим измерения;
   - служебные логи.

6. Отдельно скрипт ищет бинарные числовые записи RheoWin. В вашем тестовом
   файле таблица хранится сериями `float32` по 15 точек.

7. Найденные числовые каналы сопоставляются с колонками:
   - время;
   - `Tau`;
   - `gamma`;
   - температура;
   - частота;
   - дополнительные каналы.

8. Для каждой точки дополнительно считается `complex_modulus_pa_est = Tau/gamma`.

9. После обработки всех файлов создается одна Excel-книга.

10. В книгу добавляются только практические листы:
    - `data`;
    - `instrument_metadata`;
    - `logs`;
    - опционально `numeric_blocks`.

11. Таблицы форматируются:
    - закрепляется первая строка;
    - включается автофильтр;
    - заголовки выделяются цветом;
    - ширина колонок подгоняется под содержимое.

12. Итоговый `.xlsx` сохраняется по пути из `-o`. Если `-o` не указан, файл
    создается автоматически с датой и временем в имени.
