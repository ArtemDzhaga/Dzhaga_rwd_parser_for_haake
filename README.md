# Локальный CLI для HAAKE RheoWin `.rwd`

Скрипт добавляет данные из одного или нескольких `.rwd` файлов в выбранный лист
Excel-шаблона.

Физические названия каналов берутся из официального ASCII-экспорта RheoWin.
Скрипт не угадывает соответствия: он подтверждает их сравнением значений ASCII
и калибровочного `.rwd`.

## Что передать на вход

Аргументы передаются строго в таком порядке:

1. Excel-шаблон `.xlsx`.
2. Эталонный ASCII-экспорт RheoWin `.txt`, `.asc` или `.csv`.
3. Один или несколько `.rwd` файлов либо папок с `.rwd`.
4. Название листа Excel через `--sheet`.

Среди `.rwd` обязательно должен присутствовать файл, из которого был создан
ASCII-экспорт. Остальные `.rwd` могут быть другими экспериментами с такой же
структурой Job Manager.

## Пример запуска

```bash
cd /Users/artem/Desktop/Output/iam_ras/haake_parser
python3 haake_cli.py \
  "/Users/artem/Desktop/template.xlsx" \
  "/Users/artem/Desktop/calibration.txt" \
  "/Users/artem/Desktop/calibration.rwd" \
  "/Users/artem/Desktop/experiment_02.rwd" \
  "/Users/artem/Desktop/experiment_03.rwd" \
  --sheet "input_data" \
  -o "/Users/artem/Desktop/result.xlsx"
```

Можно перетащить файлы в терминал мышкой после ввода:

```bash
python3 haake_cli.py 
```

## Папки с `.rwd`

Вместо списка `.rwd` можно передать папку:

```bash
python3 haake_cli.py template.xlsx calibration.txt ./rwd_files --sheet input_data -o result.xlsx
```

Для поиска во вложенных папках:

```bash
python3 haake_cli.py template.xlsx calibration.txt ./rwd_files --recursive --sheet input_data -o result.xlsx
```

## Что делает скрипт

1. Открывает Excel-шаблон.
2. Проверяет, что лист из `--sheet` существует.
3. Читает официальный ASCII-экспорт RheoWin.
4. Извлекает из ASCII заголовки и эталонные числовые столбцы:
   - `t in s`;
   - `t_seg in s`;
   - `Tau in Pa`;
   - `G' in Pa`;
   - `G'' in Pa`;
   - `|Eta*| in Pas`;
   - `Gamma in -`;
   - `f in Hz`;
   - `T in °C`.
5. Читает бинарные массивы `float32` из каждого `.rwd`.
6. Находит среди переданных `.rwd` калибровочный файл, значения которого
   совпадают с ASCII.
7. Для каждой подписанной колонки ASCII определяет конкретный бинарный канал.
8. Проверяет, что остальные `.rwd` содержат такие же каналы и количество точек.
9. Разбирает имена `.rwd`, группирует файлы по составу и напряжению `U=...`.
10. Внутри каждого общего блока сортирует `.rwd` по приписке `freq=...`.
11. Добавляет сформированные блоки ниже существующих данных выбранного листа.
12. Создает или обновляет лист `instrument_metadata`.
13. Сохраняет новую Excel-книгу. Исходный шаблон не изменяется.

## Структура Excel

### Выбранный лист

Например, `input_data`.

Новые данные добавляются ниже уже заполненных таблиц.

Имя файла:

```text
SCTNA_60_PyrO_40_Katal68_2_freq=01hz_U=1500v_30_points_ver1.rwd
```

разбирается на общий блок:

```text
SCTNA_60_PyrO_40_Katal68_2_U=1500v
```

и частотный подблок:

```text
01hz
```

Структура каждого общего блока:

1. В отдельной строке, начиная с ячейки `B`, записывается имя общего блока.
2. Строкой ниже располагаются подблоки частот: `01hz`, `05hz`, `10hz` и так
   далее.
3. Еще одной строкой ниже для каждой частоты повторяются шесть рабочих колонок:
   - `t_seg in s`;
   - `Tau in Pa`;
   - `G' in Pa`;
   - `G" in Pa`;
   - `|eta*| in Pas`;
   - `gamma`.
4. Левая колонка каждого частотного подблока содержит номера точек:
   - `1|1`;
   - `1|2`;
   - `1|3`;
   - и так далее.
5. Далее записываются фактические значения из бинарных массивов `.rwd`.

Каждый частотный подблок занимает семь колонок: колонку номера точки и шесть
рабочих колонок.

Если общий блок с таким именем уже существует на выбранном листе, скрипт
остановится, чтобы не создавать незаметные дубликаты.

### `instrument_metadata`

Метаданные прибора:

- первая колонка: название поля;
- следующие колонки: отдельные `.rwd`;
- файлы расположены в алфавитном порядке.

### Скрытый лист `parser_mapping`

Техническая таблица доказательно найденных соответствий:

```text
официальный заголовок ASCII -> code_1, code_2, dtype, occurrence
```

Лист скрыт, чтобы не мешать работе. При необходимости его можно отобразить в
Excel.

## Диагностический режим

Для полного аудита исходных байтов:

```bash
python3 haake_cli.py template.xlsx calibration.txt ./rwd_files \
  --sheet input_data \
  --include-diagnostics \
  -o result.xlsx
```

Добавится лист `raw_values` с hex-смещениями и исходными значениями `float32`.

## Когда нужен новый ASCII

Новый ASCII нужен только при изменении структуры данных:

- используется другой Job Manager-скрипт;
- изменился набор экспортируемых величин;
- изменился порядок или состав бинарных каналов;
- изменилось количество точек эксперимента.

Для повторной обработки файлов той же структуры можно использовать один и тот
же калибровочный ASCII и соответствующий ему `.rwd`.

## Сообщение об ошибке калибровки

Если ASCII не соответствует ни одному переданному `.rwd`, скрипт остановится.

Это означает, что нужно добавить в команду тот `.rwd`, из которого был сделан
ASCII-экспорт. Имена обычно совпадают:

```text
experiment.txt
experiment.rwd
```
