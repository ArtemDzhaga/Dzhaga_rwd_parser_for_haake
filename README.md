# Локальный CLI для HAAKE RheoWin `.rwd`

Скрипт добавляет данные из одного или нескольких `.rwd` файлов в выбранный лист
Excel-шаблона.

Физические названия каналов берутся из официального ASCII-экспорта RheoWin.
ASCII нужен как описание структуры: какие колонки есть, сколько строк в
частотном сегменте и какие частотные сегменты ожидаются. Значения берутся из
переданных `.rwd` файлов; ASCII не обязан быть экспортирован из этих же `.rwd`.

Графики в Excel больше не строятся и не обновляются. Для графиков используется
отдельный скрипт `haake_plot.py` на `matplotlib`: так проще получить нормальные
логарифмические оси, легенду и экспорт в PNG/SVG/PDF.

## Структура проекта

```text
haake_parser/
├── haake_cli.py              # запуск импорта .rwd в Excel
├── haake_plot.py             # запуск построения графиков
├── haake_rheo/
│   ├── __init__.py
│   ├── excel_import.py       # парсинг ASCII/.rwd, структурная карта, запись в Excel
│   └── plot.py               # чтение Excel и построение matplotlib-графиков
├── README.md
├── requirements.txt
└── .gitignore
```

Корневые `haake_cli.py` и `haake_plot.py` оставлены короткими entrypoint-файлами.
Основная логика находится в пакете `haake_rheo`.

## Основной UX

### 1. Заполнить Excel-книгу

Запустите скрипт без аргументов:

```bash
cd /Users/artem/Desktop/Output/iam_ras/haake_parser
python3 haake_cli.py
```

Дальше мастер спросит:

1. Excel-шаблон `.xlsx`, где уже понятно, какие блоки и частоты заполнять.
2. ASCII-файл RheoWin `.txt`, `.asc` или `.csv`, по которому определяется
   структура бинарных каналов.
3. Один или несколько `.rwd` файлов либо папку с `.rwd`. Файлы могут быть
   одиночными или пакетными.
4. Название листа, куда писать данные. По умолчанию `input_data`.
5. Итоговый `.xlsx` файл, куда сохранить результат.

Файлы и папки можно перетаскивать в терминал мышкой.

Лучше писать результат в новый файл, например `may_proj_v1_4.xlsx`, а не в
открытый в Excel файл. Если Excel держит книгу открытой, рядом появляется
lock-файл вида `~$may_proj_v1_2.xlsx`, и перезапись того же имени может
запутать проверку.

В конце работы скрипт печатает статистику времени:

```text
Время выполнения:
  - Валидация входных путей: ...
  - Чтение ASCII-структуры: ...
  - Чтение .rwd файлов: ...
  - Построение структурной карты: ...
  - Загрузка Excel-шаблона: ...
  - Заполнение Excel-книги: ...
  - Сохранение Excel-книги: ...
  - Итого: ...
```

### 2. Построить графики

Графики строятся отдельным Python-скриптом:

```bash
python3 haake_plot.py
```

Мастер спросит:

1. Excel-книгу после парсинга.
2. Лист с данными. По умолчанию `output_data`.
3. Папку для сохранения графиков.
4. Сколько первых точек брать из каждого подблока.
5. Порядок полинома.
6. Форматы файлов. По умолчанию `png`.
7. Метрики.
8. Частоты. Если оставить пусто, будут построены все найденные частоты.

После построения графиков скрипт также печатает статистику времени:

```text
Время выполнения:
  - Валидация аргументов: ...
  - Загрузка библиотек: ...
  - Чтение данных Excel: ...
  - Построение и сохранение графиков: ...
  - Итого: ...
```

## Старый режим командной строки

Старый запуск с аргументами тоже работает. Аргументы передаются в таком порядке:

1. Excel-шаблон `.xlsx`.
2. ASCII-экспорт RheoWin `.txt`, `.asc` или `.csv`, задающий структуру колонок.
3. Один или несколько `.rwd` файлов либо папок с `.rwd`.
4. Название листа Excel через `--sheet`.

ASCII не обязан совпадать с переданными `.rwd` по значениям. Он задает только
структуру: порядок рабочих колонок, количество строк в сегменте и набор
частотных сегментов. Поэтому можно выбрать ASCII от `1500v`, а затем передать
`.rwd` от `2000v`, `2500v` и других экспериментов той же структуры Job Manager.

Если `.rwd` содержит не все частоты из ASCII, скрипт заполнит только реально
найденные частоты. Например, при ASCII `0.1-100 Гц` файл `1-100 Гц` не будет
заполнять пустые подблоки `0,1` и `0,5 Гц`.

ASCII может быть одиночным или пакетным:

- одиночный ASCII: один частотный подблок, например `01hz`, 30 строк;
- пакетный ASCII: несколько подблоков, например 10 частот по 30 строк.

Для пакетных файлов скрипт сам читает сегменты из ASCII, определяет фактические
частоты по колонке `f in Hz` и раскладывает данные по соответствующим
частотным подблокам Excel.

### Пример запуска

```bash
cd /Users/artem/Desktop/Output/iam_ras/haake_parser
python3 haake_cli.py \
  "/Users/artem/Desktop/template.xlsx" \
  "/Users/artem/Desktop/structure.txt" \
  "/Users/artem/Desktop/experiment_02.rwd" \
  "/Users/artem/Desktop/experiment_03.rwd" \
  --sheet "input_data" \
  -o "/Users/artem/Desktop/result.xlsx"
```

## Папки с `.rwd`

Вместо списка `.rwd` можно передать папку:

```bash
python3 haake_cli.py template.xlsx structure.txt ./rwd_files --sheet input_data -o result.xlsx
```

Для поиска во вложенных папках:

```bash
python3 haake_cli.py template.xlsx structure.txt ./rwd_files --recursive --sheet input_data -o result.xlsx
```

## Что делает скрипт

1. Открывает Excel-шаблон.
2. Проверяет, что лист из `--sheet` существует.
3. Читает официальный ASCII-экспорт RheoWin.
4. Извлекает из ASCII заголовки, количество строк в сегменте и список частот:
   - `t in s`;
   - `t_seg in s`;
   - `Tau in Pa`;
   - `G' in Pa`;
   - `G'' in Pa`;
   - `|Eta*| in Pas`;
   - `Gamma in -`;
   - `f in Hz`;
   - `T in °C`.
5. Читает бинарные массивы `float32` из каждого `.rwd`.
6. По заголовкам ASCII строит структурную карту каналов RheoWin.
7. Для пакетного ASCII рассчитывает смещения каждого частотного сегмента.
8. Проверяет, какие частотные сегменты реально присутствуют в каждом `.rwd`.
    Например, файл `freq=from_1hz_to_100hz` может заполнить только 1-100 Гц,
    не затрагивая 0,1 и 0,5 Гц.
9. Разбирает имена `.rwd`, группирует файлы по составу и напряжению `U=...`.
10. Внутри каждого общего блока сортирует `.rwd` по частоте.
11. Добавляет данные в существующие блоки шаблона или ниже существующих данных.
12. Создает или обновляет лист `instrument_metadata`.
13. Сохраняет новую Excel-книгу. Исходный шаблон не изменяется.

Скрипт `haake_cli.py` не создает Excel-графики, не меняет лист `freq_diagrams`
и не создает скрытые листы с кривыми для диаграмм.

## Структура Excel

### Выбранный лист

Например, `input_data`.

Новые данные добавляются ниже уже заполненных таблиц.

Имя файла:

```text
SCTNA_60_PyrO_40_Katal68_2_freq=01hz_U=1500v_30_points_ver1.rwd
```

разбирается на общий блок:

```text
SCTNA_60_PyrO_40_Katal68_2_U=1500v
```

и частотный подблок:

```text
01hz
```

Структура каждого общего блока:

1. В отдельной строке, начиная с ячейки `B`, записывается имя общего блока.
2. Строкой ниже располагаются подблоки частот: `01hz`, `05hz`, `10hz` и так
   далее.
3. Еще одной строкой ниже для каждой частоты повторяются шесть рабочих колонок:
   - `t_seg in s`;
   - `Tau in Pa`;
   - `G' in Pa`;
   - `G" in Pa`;
   - `|eta*| in Pas`;
   - `gamma`.
4. Левая колонка каждого частотного подблока содержит номера точек:
   - `1|1`;
   - `1|2`;
   - `1|3`;
   - и так далее.
5. Далее записываются фактические значения из бинарных массивов `.rwd`.

Каждый частотный подблок занимает семь колонок: колонку номера точки и шесть
рабочих колонок.

Если общий блок с таким именем уже существует на выбранном листе, скрипт
остановится, чтобы не создавать незаметные дубликаты.

### `instrument_metadata`

Метаданные прибора:

- первая колонка: название поля;
- следующие колонки: отдельные `.rwd`;
- файлы расположены в алфавитном порядке.

### Скрытый лист `parser_mapping`

Техническая таблица структурных соответствий:

```text
официальный заголовок ASCII -> code_1, code_2, dtype, occurrence, segment_stride
```

Лист скрыт, чтобы не мешать работе. При необходимости его можно отобразить в
Excel.

## Диагностический режим

Для полного аудита исходных байтов:

```bash
python3 haake_cli.py template.xlsx structure.txt ./rwd_files \
  --sheet input_data \
  --include-diagnostics \
  -o result.xlsx
```

Добавится лист `raw_values` с hex-смещениями и исходными значениями `float32`.

## Построение графиков через Python

После создания Excel-книги графики можно строить интерактивно:

```bash
python3 haake_plot.py
```

Или старой командой с аргументами:

```bash
python3 haake_plot.py \
  "/Users/artem/Desktop/Output/iam_ras/haake_parser/output/may_proj_v1_2.xlsx" \
  --sheet "output_data" \
  -o "/Users/artem/Desktop/Output/iam_ras/haake_parser/output/plots" \
  --points 20 \
  --poly-order 3
```

Что делает `haake_plot.py`:

1. Читает готовую Excel-книгу.
2. Читает лист `output_data`, где уже подготовлены первые 20 точек и пары
   `Gamma_* / G'_*`, `Gamma_* / G''_*`, `Gamma_* / |Eta*|_*`, `tan_δ_*`.
3. Находит частотные блоки: `0,1 Hz`, `0,5 Hz`, `1 Hz` и т.д.
4. Берет первые точки из каждого подблока. По умолчанию берутся первые 20 точек.
5. Строит четыре типа графиков:
   - `G' / gamma`;
   - `G" / gamma`;
   - `|eta*| / gamma`;
   - `tan delta / gamma`, где `tan delta = G" / G'`.
6. Для обеих осей включает логарифмическую шкалу.
7. Рисует точки разными цветами и маркерами.
8. Добавляет полиномиальную кривую в координатах `log10(gamma)` и `log10(y)`.
9. В легенду добавляет только экспериментальные точки. Полиномиальные кривые в
   легенду не выводятся.
10. Сохраняет отдельные PNG-файлы графиков в указанную папку.

Если библиотеки не установлены:

```bash
python3 -m pip install matplotlib numpy openpyxl
```

### Основные настройки графиков

Количество точек:

```bash
--points 20
```

Порядок полинома:

```bash
--poly-order 3
```

Если кривая выглядит слишком жесткой, попробуйте `--poly-order 2`. Если нужно
больше гибкости, попробуйте `--poly-order 4`, но высокий порядок легче
переобучается.

Отключить полиномиальные кривые и оставить только точки:

```bash
--no-fit
```

Построить только часть метрик:

```bash
--metrics g_prime g_double_prime
```

Построить только выбранные частоты:

```bash
--frequencies 0.1 0.5 1
```

По умолчанию сохраняется только PNG. Если нужен векторный формат, можно
добавить SVG или PDF явно:

```bash
--formats png svg pdf
```

Для быстрой проверки удобен `png`. Для дальнейшей правки в векторных редакторах
лучше использовать `svg` или `pdf`.

## Когда нужен новый ASCII

Новый ASCII нужен только при изменении структуры данных:

- используется другой Job Manager-скрипт;
- изменился набор экспортируемых величин;
- изменился порядок или состав бинарных каналов;
- изменилось количество точек эксперимента.
- одиночные `.rwd` и пакетные `.rwd` смешиваются без общего пакетного шаблона
  сегментов.

Для повторной обработки файлов той же структуры можно использовать один и тот
же ASCII. Передавать `.rwd`, из которого был сделан ASCII, не требуется.

## Если структура не подходит

Если в `.rwd` нет ожидаемых бинарных каналов, скрипт остановится.

Это означает, что файл был получен другой программой Job Manager или другим
составом экспортируемых величин. В этом случае нужен новый ASCII-экспорт,
который описывает именно эту структуру:

```text
new_structure.txt
experiment_01.rwd
experiment_02.rwd
```
